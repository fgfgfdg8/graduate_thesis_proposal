\section{预期的成果}

\subsection{成果一：工具调用链风险度量与可解释评估框架（对应研究点一）}

\begin{enumerate}
  \item \textbf{统一日志Schema与任务集。}完成面向工具增强型LLM Agent的统一日志Schema设计，形成覆盖文件管理、邮件办公、数据库查询、日程管理、代码执行和IoT控制六类工具场景的低成本任务集，包含600条以上工具调用链样例，覆盖$R_0$至$R_6$全部风险类型。

  \item \textbf{多维风险指标体系与评估框架。}实现工具敏感度、权限扩散度、调用深度、不可逆操作比例、敏感数据暴露度和人工确认覆盖率六个可解释指标的自动计算，以及综合风险评分函数和指标贡献分解输出。通过对比实验（与单步权限检查、敏感工具计数等基线对比）和消融实验（各指标独立贡献分析）验证框架有效性。

  \item \textbf{可插拔评估模块。}实现能够对任意执行日志输出风险分数、指标贡献和审计报告的可插拔Python模块，支持命令行接口调用，可作为Agent部署前评估和事后审计的轻量组件。
\end{enumerate}

\subsection{成果二：行为轨迹图工具滥用检测与风险归因方法（对应研究点二）}

\begin{enumerate}
  \item \textbf{异构行为轨迹图建模方法。}提出适用于LLM Agent执行过程的异构行为轨迹图表示，将自然语言任务、计划、工具调用、工具参数、返回结果、记忆写入和最终输出统一到一个可查询的图结构中，并实现确定性图构建算法。

  \item \textbf{图模式检测与轻量分类器。}构建面向六类工具滥用模式（过度授权、敏感数据外传、不可逆操作缺少确认、参数来源污染、跨Agent委托失控、共享记忆污染）的图规则库，并结合轻量分类器实现组合风险识别。在三个合成数据集上完成与14个代理基线方法的对比实验，验证图检测相对于序列方法的优势。

  \item \textbf{风险路径定位与反事实归因。}实现Top-K风险路径定位算法和基于反事实消融的节点/边归因方法，输出包含异常类型、风险路径、关键节点、触发工具、参数来源和归因贡献的解释性审计报告。通过Path Recall@K、Hit@K和MRR等指标定量验证归因准确性。

  \item \textbf{原型应用系统。}基于上述方法设计并实现TraceGraph-Audit可插拔审计原型系统，支持日志输入、图构建、风险检测、归因分析和报告生成等功能模块，并在真实Agent执行日志上进行案例分析验证。

\end{enumerate}

\subsection{成果三：可插拔LLM Agent工具调用链审计原型系统}

基于整体研究内容，设计并实现TraceGraph-Audit可插拔审计原型系统，系统包含六大核心模块：

\begin{enumerate}
  \item \textbf{Trace Collector（轨迹采集模块）：}采集或读取Agent执行日志，规范化为统一Schema，支持多种Agent框架的日志格式。
  \item \textbf{Risk Metric Engine（风险指标引擎）：}计算工具敏感度、权限扩散度、不可逆比例、敏感暴露度等指标，输出综合风险分数和指标贡献分解。
  \item \textbf{Graph Builder（图构建模块）：}将日志转换为异构行为轨迹图，生成节点、边和特征，支持可视化输出。
  \item \textbf{Misuse Detector（滥用检测模块）：}基于图规则和轻量模型检测工具滥用风险，输出风险类型和置信度。
  \item \textbf{Attribution Analyzer（归因分析模块）：}输出Top-K风险路径、节点贡献和边贡献，生成可解释归因报告。
  \item \textbf{Audit Report Generator（审计报告生成模块）：}生成中文/英文审计报告、实验表格和可视化摘要，支持命令行和批量处理。
\end{enumerate}
