\section{预计的研究难点及解决方案}

\begin{enumerate}
  \item \textbf{数据集和标签构造难度。}公开Agent安全数据集中，多数侧重提示注入或最终攻击成功率，未必包含完整的工具调用链、权限标签和风险路径标签。若完全依赖公开数据，可能无法支撑本课题的指标计算和归因实验。拟采用“公开基准思想+自建可控任务集”的方式：基于AgentDojo\cite{balunovic2024agentdojo}和AgentCanary\cite{li2026agentcanary}两个公开基准采集执行轨迹，标签由A1--A6攻击链规则自动生成后再进行人工抽查，保证可复现性和标注成本可控。\texttt{gold\_path}与\texttt{gold\_node}归因标签通过逐步复现攻击步骤确定关键触发节点。

  \item \textbf{七维风险因子权重整定的平衡与稳定性。}七个风险因子（$S_{\text{tool}},$ $D_{\text{call}},$ $P_{\text{diff}},$ $E_{\text{leak}},$ $I_{\text{irrev}},$ $H_{\text{cov}},$ $N_{\text{need}}$）的量纲和动态范围不同，且不同部署场景下各维度重要性存在差异，直接加权组合可能导致某些信号主导综合风险评分。拟采用“威胁先验初始化—弱监督标签生成—约束优化校准”四步整定策略：先验权重根据威胁后果不可恢复程度设置，约束优化同时施加二分类损失、排序损失和先验正则，防止权重偏离先验过远；在验证集上搜索最优阈值组并报告ECE和Brier Score，验证校准质量。

  \item \textbf{行为轨迹图构建的完整性与$V_{\text{guard}}$/$E_{\text{defend}}$接入。}不同Agent执行框架的日志字段存在缺失或不一致，可能导致构建的轨迹图缺失部分依赖边，影响检测和归因的准确性；同时，防御节点$V_{\text{guard}}$和防御边$E_{\text{defend}}$需要与Defense Guard中间层的实际动作保持一致，存在接口对齐挑战。拟设计鲁棒的图构建规则：对于缺失字段，采用默认值填充并标注不确定性；对于参数来源不明的情况，增加“unknown\_source”节点；$V_{\text{guard}}$在Pre-call/In-call/Post-call 防御动作触发时自动注入，$E_{\text{defend}}$指向被防御的工具节点，保持接口一致性。

  \item \textbf{最小干预防御动作选择的效用-安全权衡。}防御动作优化公式（式~\ref{eq:min_intervention}）中$\lambda_s, \lambda_u, \lambda_c$的权衡系数设定直接影响DSR（防御成功率）与UPR（效用保持率）的平衡；不同场景下最优权衡系数不同，难以统一设定。拟在AgentDojo和AgentCanary验证集上进行网格搜索，绘制DSR-UPR帕累托前沿曲线，选取帕累托最优点对应的权衡系数；同时提供场景自适应接口，允许用户根据业务需求调整$\lambda_s$（安全优先）或$\lambda_u$（效用优先）。

  \item \textbf{图规则库覆盖完备性验证。}8条核心规则（Rule-1至Rule-8）能否覆盖A1--A6六类攻击的全部变体，以及规则组合是否存在误报，需要系统性验证。拟通过消融实验分别测试每条规则的精确率和召回率贡献；对于规则未覆盖的样本，分析其路径模式并补充规则或交由第二级分类器处理；同时统计第一级规则在AgentDojo和AgentCanary上的激活比例，验证规则库的实际覆盖范围。

  \item \textbf{论文创新性表达与工程实现的平衡。}单纯做指标汇总或日志可视化可能被认为工程性较强、方法创新不足。拟将TARC-Guard的七维风险因子+约束优化校准+L0--L4分级防御、TraceGraph-Guard的带防御节点轨迹图+类型化路径特征+图分数序列先验+最小干预防御、以及Defense Guard可插拔中间层作为三个独立的方法贡献；通过消融实验分别验证各组件的贡献，通过DSR/UPR指标首次量化防御有效性与任务效用的联合评估，凸显方法创新价值。
\end{enumerate}
