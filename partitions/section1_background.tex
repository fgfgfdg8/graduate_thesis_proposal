\section{研究背景与意义}

\subsection{研究背景}

近年来，随着深度学习技术的飞速突破，大语言模型（Large Language Model，LLM）正经历着一场深刻的范式变革：从单纯依赖统计规律生成连贯文本的被动工具，逐步演化为具备主动任务规划、工具调用、环境交互及多轮反馈修正能力的智能体（Agent）。据Precedence Research权威统计，2025年全球AI Agent市场规模已飙升至79.2亿美元，且预计至2035年将爆发式增长至2946.6亿美元，年均复合增长率（CAGR）高达43.57\%\cite{precedence2025aiagents}。这一增长趋势得到了Gartner的强力印证，其预测指出，到2028年，将有33\%的生成式AI交互彻底摒弃传统的问答模式，转而采用Agent式的自主决策模式\cite{gartner2025agentic}。在产业实践层面，国际科技巨头与国内领军企业已形成百舸争流的竞争格局：OpenAI推出的Agents SDK与Codex、Anthropic的Claude Code、Google的Gemini Agent以及微软的Copilot Studio等产品不断刷新技术上限；与此同时，阿里巴巴百炼、字节跳动Coze、智谱AI AutoGLM等国内平台也迅速崛起，共同推动Agent技术从理论走向应用。在开源生态中，LangChain\cite{langchain}、MetaGPT\cite{hong2024metagpt}、CrewAI、AutoGen\cite{wu2024autogen}等开发框架的广泛应用，极大地降低了Agent开发门槛，标志着该技术已从学术实验室的原型阶段迈向大规模商业部署阶段，并深度渗透至办公自动化、软件工程、网络安全运维及复杂数据分析等关键业务领域\cite{wang2024survey,xi2025rise}。

从技术演进逻辑来看，LLM从文本生成器向智能体的角色跃迁并非一蹴而就，而是经历了数个具有里程碑意义的关键节点。首先，Transformer架构\cite{vaswani2017attention}的提出奠定了序列建模的高效计算基础；随后，GPT-3\cite{brown2020language}的出现验证了大规模预训练模型在少样本场景下的惊人泛化能力；InstructGPT\cite{ouyang2022training}通过指令微调与基于人类反馈的强化学习，成功实现了模型行为与人类意图的初步对齐。而GPT-4\cite{openai2023gpt4}的发布则是技术分水岭，它首次原生支持函数调用机制，使LLM能够直接生成符合语法规范的结构化工具调用请求，并精准解析返回结果，这标志着LLM从单纯的“文本生成器”向具备实际操作能力的“任务执行器”实现了质的转变。在Agent形态层面，Weng\cite{weng2023agent}提出了著名的架构公式：Agent = 大模型 + 规划 + 工具调用 + 记忆（如\autoref{fig:agent-overview}所示），这一公式清晰界定了现代LLM Agent的四大核心支柱，成为了业界的共识基础。LLM Agent区别于传统软件智能体的关键在于，它以自然语言作为统一的交互接口，通过灵活的提示词驱动复杂的推理过程与实际行动，而非依赖僵硬的预编程规则\cite{wang2024survey}。在此框架下，工具调用不仅是感知环境的触角，更是Agent作用于外部世界的核心执行通道。在交互范式上，Chain-of-Thoughts\cite{bosma2022chainofthought}赋予模型深层推理能力，ReAct\cite{yao2023react}实现了推理与行动的交替循环，Reflexion\cite{shinn2023reflexion}则引入了自我反思机制。这些范式的迭代不仅奠定了现代LLM Agent的交互基础，更使工具调用成为了其区别于传统语言模型的核心能力标志。正是这种能力，让Agent具备了影响并改变外部环境的实质性力量，使其无限接近于一个类似人类的智能系统——能够自主从环境中获取信息、与复杂环境进行实时交互、根据反馈进行反思并持续改进。

然而，技术的飞跃往往伴随着风险的重构。工具调用能力的引入，在极大拓展Agent任务边界与应用可能性的同时，也将传统语言模型的安全问题从封闭、可控的文本域推向了开放、不可预测的交互域。传统LLM的安全研究主要聚焦于“有害内容生成”的攻防博弈，其攻击面严格限定在自然语言层面，即提示词与生成文本之间；而工具调用型Agent的安全风险则有着本质的不同：Agent不仅具备理解指令的认知能力，更拥有通过工具执行代码、访问文件系统、操控数据库乃至操作物理设备的执行能力。这种能力使LLM Agent的安全问题从单纯关注“模型是否生成有害文本”，急剧扩展为必须严控“语言决策是否触发具有现实破坏性的外部动作”。更为严峻的是，自动化程序从“规则驱动”向“语言驱动”的范式转变，意味着智能体的行为决策由动态的语义而非静态的语法决定，这导致传统的基于静态规则的访问控制与权限管理机制面临根本性的适用性挑战\cite{chu2026systematic}。这一安全风险的结构性缺陷首次被系统性揭示，可追溯至Greshake等\cite{greshake2023notwhat}于2023年披露的“间接提示注入攻击”。研究表明，攻击者无需直接接触模型接口，只需将恶意指令精心嵌入网页、电子邮件或文档等外部数据源中，一旦Agent通过工具读取并解析这些看似无害的数据，恶意指令即可悄无声息地注入Agent的推理上下文，进而劫持其后续行为。这一发现从根本上暴露了工具调用场景中“数据与指令边界模糊”的致命安全隐患。此后，随着LLM Agent工具生态的日益繁荣与复杂化，攻击向量也随之迅速扩散与进化：攻击者不仅可以劫持工具的选择决策\cite{shi2026pitoolselection}，诱导Agent调用恶意工具，还可通过社会工程学手段诱导Agent请求远超任务实际所需的敏感权限\cite{zhang2026evaluatingprivilege}。更为隐蔽的是，攻击者能够利用多步工具调用的逻辑链条，将多个看似合法的子任务进行恶意组合，最终达成恶意目标\cite{kothamasu2026hidden}。因此，LLM Agent的安全问题已不再是单点的提示注入防御，而是演变为覆盖工具选择合理性、权限最小化原则、多步调用逻辑组合的系统性安全挑战。

综合上述背景，面对日益复杂的攻击手段和不断扩大的风险边界，针对工具调用链的安全风险感知、精准归因与有效防御，已成为Agent安全领域亟待深入研究的核心课题。本研究致力于提出一种面向LLM Agent工具调用链的可解释风险度量与威胁归因方法，旨在解决当前安全防护“黑盒化”与“被动性”的痛点。具体而言，本研究将工具敏感性、权限边界、调用深度、操作不可逆性等关键要素纳入统一的风险度量框架，对工具调用链中可能产生的组合性威胁进行分级量化评估；在此基础上，借鉴系统安全领域的溯源图思想，构建一种能够适应Agent执行语义的异构轨迹图表示方法，以实现对跨步骤风险路径的自动识别与基于证据的可解释威胁归因。本研究期望为LLM Agent工具调用链的可解释安全性保障提供坚实的理论支撑与切实可行的工程实践指导。


\subsection{研究意义}

\begin{enumerate}
  \item 本研究从大语言模型智能体工具调用链出发，聚焦Agent的“过程级行为安全”，提出可解释风险度量体系，将工具敏感性、权限边界、调用深度、不可逆性和人工确认等因素纳入统一框架，对工具调用链的组合性威胁风险进行分级量化和可解释归因；在此基础上基础上，进一步将Agent执行过程中的工具上下文元素之间的依赖关系进行形式化图表达，为工具调用链风险检测和归因提供结构化基础，有助于弥补现有研究中“重防御结果、轻过程感知”的不足。
  \item 本研究将系统安全领域的溯源图思想迁移至LLM Agent场景，提出适应Agent执行语义的异构轨迹图表示，为Agent可观测性和执行溯源研究提供新的方法论视角，从而为Agent安全从“攻防对抗”向“过程审计、威胁归因”的防御范式演进提供一种可能的思路，以弥补现有工作在跨步骤风险路径识别、可解释威胁归因和最小干预防御方面的局限性。
  \item 本研究旨在设计一个轻量级的位于LLM Agent执行层和工具层之间的风险审计与防御编排层，通过跟踪不可信数据、工具权限、敏感数据流和不可逆副作用，对 Agent 的每一步工具调用进行细粒度风险评估，并根据风险类型执行阻断、净化、模拟执行、请求确认、回滚或阻断等防御动作，为Agent系统部署前评估、运行时监控和事后审计提供一种可能的方案。
\end{enumerate}
